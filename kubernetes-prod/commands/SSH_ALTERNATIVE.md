# Альтернативные способы подключения к ВМ

Если SSH не работает, используйте эти методы:

## Способ 1: Веб-консоль Yandex Cloud

1. Откройте [Yandex Cloud Console](https://console.cloud.yandex.ru/)
2. Перейдите в **Compute Cloud** → **Виртуальные машины**
3. Найдите нужную ВМ (например, `k8s-master`)
4. Нажмите на ВМ, затем кнопку **"Подключиться"**
5. Выберите **"SSH"** или **"Веб-терминал"**
6. Используйте веб-интерфейс для работы

## Способ 2: Добавить SSH ключ через веб-консоль

1. Откройте ВМ в веб-консоли
2. Перейдите на вкладку **"SSH ключи"**
3. Нажмите **"Добавить ключ"**
4. Вставьте ваш публичный SSH ключ:
   ```bash
   cat ~/.ssh/id_rsa.pub
   ```
5. Сохраните и попробуйте подключиться снова

## Способ 3: Использовать Serial Console

1. В веб-консоли ВМ нажмите **"Подключиться"** → **"Serial Console"**
2. Войдите с учетными данными:
   - Пользователь: `ubuntu`
   - Пароль: (если был установлен при создании)

## Способ 4: Создать новый SSH ключ на ВМ

Если у вас есть доступ через веб-консоль:

```bash
# На ВМ выполните:
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa -N ""
cat ~/.ssh/id_rsa.pub
```

Затем скопируйте публичный ключ и добавьте его в `~/.ssh/authorized_keys` на других ВМ.

## Способ 5: Использовать yc для выполнения команд

Если yc CLI настроен, можно выполнять команды удаленно:

```bash
# Выполнить команду на ВМ через yc
yc compute instance execute k8s-master --command "hostname && whoami"
```

## Рекомендация

**Используйте веб-консоль Yandex Cloud** - это самый простой и надежный способ для начальной настройки.

После подключения через веб-консоль выполните команды из:
- `commands/MASTER_COMMANDS.sh` - для master-ноды
- `commands/01-prepare-master.md` - пошаговые инструкции
